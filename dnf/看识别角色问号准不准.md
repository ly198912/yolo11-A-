# 看识别角色问号准不准

这份说明用来判断程序有没有正确识别小地图上的角色位置和问号位置。

## 先打开小地图调试窗口

启动程序前，在 PowerShell 里设置：

```powershell
$env:DNF_DEBUG_MINIMAP="1"
python -m dnf.main
```

程序会多出一个 `dnf-minimap-debug` 窗口。

这个窗口里的标记含义：

- `P` = 角色当前位置
- `Q` = 问号位置
- `B` = boss
- `E` = 精英/特殊房
- `D` = 下楼
- 白框和灰线 = 程序认为的小地图房间网格

## 最直观的判断方法

看 `dnf-minimap-debug` 窗口：

- `P` 应该压在小地图上的角色图标位置。
- `Q` 应该压在小地图上的真实问号位置。
- 白框和灰线应该刚好套住小地图里的房间格子。

如果游戏小地图里的问号在右上，但 debug 窗口里的 `Q` 跑到左边、下面，说明问号识别或小地图网格不准。

如果游戏小地图里的角色在某个房间，但 debug 窗口里的 `P` 跑到别的格子，说明角色识别或小地图网格不准。

## 看日志怎么判断

主循环会打印类似这样的日志：

```text
route: map=generic, current=(1, 6), boss=(1, 7), query=(2, 4), elite=(1, 7), down=(2, 5), target=query@(2, 4), direction=LEFT, door=None, scores={'hero': 0.903, 'boss': 0.8963, 'elite': 0.8696, 'query': 0.9092, 'query1': 0.9161, 'down': 0.7618}
```

重点看这些字段：

- `current=(1, 6)`：程序认为角色当前所在房间。
- `query=(2, 4)`：程序认为问号所在房间。
- `target=query@(2, 4)`：程序当前选择的目标是问号房。
- `direction=LEFT`：程序根据当前房间和目标房间算出来的移动方向。
- `scores`：模板匹配分数，里面的 `hero`、`query`、`query1` 最重要。

## 房间坐标怎么看

坐标格式是：

```text
(行, 列)
```

规则：

- 第一个数字是行，越大越靠下。
- 第二个数字是列，越大越靠右。

例如：

```text
current=(1, 6)
query=(1, 7)
```

这表示问号在角色右边一格。

再比如：

```text
current=(1, 6)
query=(2, 6)
```

这表示问号在角色下面一格。

## 什么情况说明识别错了

如果你肉眼看到问号在右边，但日志显示：

```text
current=(1, 6)
query=(2, 4)
direction=LEFT
```

这就是不对的。因为 `query=(2, 4)` 的列数 `4` 比角色列数 `6` 小，程序会认为问号在左边。

如果肉眼看到问号在右上，但日志一直算成 `LEFT`、`DOWN`、`LEFT_DOWN`，也说明小地图识别、房间网格或地图类型有问题。

## scores 分数怎么看

重点看：

```text
scores={'hero': 0.903, 'query': 0.9092, 'query1': 0.9161}
```

一般判断：

- `hero` 大于 `0.8`：角色图标通常识别比较稳。
- `query` 或 `query1` 大于 `0.8`：问号通常识别比较稳。
- `query` / `query1` 在 `0.5 ~ 0.65`：可能会误识别，需要小心。
- `query=None`：程序没有识别到问号。
- `target` 不是 `query@...`：程序当前没有把问号当作目标，可能是问号没识别到，或者被 boss、精英、下楼等目标优先级影响。

## 最可靠的检查顺序

1. 开 `DNF_DEBUG_MINIMAP=1`。
2. 看 debug 窗口里的 `P` 是否压在真实角色图标上。
3. 看 debug 窗口里的 `Q` 是否压在真实问号上。
4. 看白框和灰线是否刚好套住房间格子。
5. 看日志里的 `current` 和 `query` 是否符合肉眼看到的位置。
6. 看 `direction` 是否符合从角色到问号的方向。

只要 debug 窗口里的 `P`、`Q` 位置对，日志里的 `current`、`query` 也对，程序就基本正确识别了角色和问号。

如果 `P`、`Q` 位置对，但 `current`、`query` 坐标不对，优先检查小地图房间网格，也就是 `room_rect_*`。

如果 `P`、`Q` 位置本身就不对，优先检查模板、阈值、小地图截图区域，也就是 `dnf/res/` 里的模板图和 `crop_rect_*`。
