# 新增小地图配置和接入说明

这份说明讲的是：如果你想新增一个地图，比如 `haibolun`，应该改哪些文件、怎么调小地图截图区域、怎么让主程序使用新增地图。

## 1. 核心结论

新增地图主要改这个文件：

```text
dnf/map_specs.py
```

主程序不需要额外写 `import`，因为 `dnf/main.py` 里的 `MiniMapNavigator` 会自动读取 `MAP_SPECS`。

只要你把新地图加到 `MAP_SPECS` 里，然后运行时指定地图名，主程序就能使用：

```powershell
$env:DNF_MAP_NAME="haibolun"
python -m dnf.main
```

如果只是想看小地图区域准不准，不想让角色移动，用这个单独调试脚本：

```text
dnf/minimap_grid_preview.py
```

它只截图、画网格、画 `P/Q/B/E/D` 标记，不会按键、不检测怪、不移动。

## 2. 新增 MapSpec 要写在哪里

打开：

```text
dnf/map_specs.py
```

在 `MAP_SPECS` 里面新增一项，例如：

```python
"haibolun": MapSpec(
    name="haibolun",
    crop_rect_1067=(966, 52, 1056, 124),
    crop_rect_800=(640, 57, 780, 137),
    minimap_width=140,
    minimap_height=80,
    rows=4,
    cols=5,
    room_grid=_all_walkable(4, 5),
    room_rect_800=(665, 82, 779, 136),
),
```

注意：你原本写的这个版本也可以作为初始模板：

```python
"haibolun": MapSpec(
    name="haibolun",
    crop_rect_1067=(966, 52, 1056, 124),
    crop_rect_800=None,
    minimap_width=90,
    minimap_height=72,
    rows=4,
    cols=5,
    room_grid=_all_walkable(4, 5),
),
```

但是如果你当前游戏是 `800x600`，`crop_rect_800=None` 就不够用。程序会优先用 800 分辨率配置，所以最好补上：

```python
crop_rect_800=(左, 上, 右, 下)
room_rect_800=(左, 上, 右, 下)
```

## 3. 每个字段是什么意思

### name

地图名字，要和 `MAP_SPECS` 的 key 一致：

```python
name="haibolun"
```

运行主程序时也用这个名字：

```powershell
$env:DNF_MAP_NAME="haibolun"
```

### crop_rect_800 / crop_rect_1067

这是“从整张游戏截图里裁出右上角小地图”的区域。

格式是：

```python
(x1, y1, x2, y2)
```

含义：

- `x1`：左边界
- `y1`：上边界
- `x2`：右边界
- `y2`：下边界

比如：

```python
crop_rect_800=(640, 57, 780, 137)
```

表示从 800x600 游戏截图里裁：

- 左边 `x=640`
- 上边 `y=57`
- 右边 `x=780`
- 下边 `y=137`

这个区域应该把整个右上角小地图包含进去。

### room_rect_800 / room_rect_1067

这是“小地图里面真正用于划房间网格的区域”。

它也是整张游戏截图坐标，不是裁剪后的小图坐标。

例如：

```python
room_rect_800=(665, 82, 779, 136)
```

它表示白色网格只画在这个区域里。

如果这个区域偏了，程序就会把角色和问号算错房间。比如真实问号在右边，但日志显示 `direction=DOWN`，通常就是这里不准。

### rows / cols

地图房间网格有几行几列。

比如海伯伦如果按 4 行 5 列算：

```python
rows=4
cols=5
room_grid=_all_walkable(4, 5)
```

### minimap_width / minimap_height

这是小地图裁剪出来后的参考宽高。

如果 `crop_rect_800=(640, 57, 780, 137)`，那么：

```text
宽度 = 780 - 640 = 140
高度 = 137 - 57 = 80
```

所以写：

```python
minimap_width=140
minimap_height=80
```

## 4. 怎么让主程序使用新增地图

主程序入口是：

```text
dnf/main.py
```

里面已经有这行：

```python
navigator = MiniMapNavigator(os.getenv("DNF_MAP_NAME", "auto"))
```

意思是：

- 如果你设置了 `DNF_MAP_NAME`，就用指定地图。
- 如果没设置，就用 `auto` 自动判断。

所以新增 `MapSpec` 后，不需要在 `main.py` 里再调用什么代码。

你只要这样运行：

```powershell
$env:DNF_MAP_NAME="haibolun"
python -m dnf.main
```

主程序就会走：

```text
dnf/main.py
  -> MiniMapNavigator("haibolun")
  -> dnf/minimap_nav.py
  -> dnf/map_specs.py 里的 MAP_SPECS["haibolun"]
```

## 5. 怎么只调试小地图，不让角色动

用这个文件：

```text
dnf/minimap_grid_preview.py
```

它的默认地图在这里：

```python
MAP_NAME = "haibolun"
```

你可以直接双击运行，也可以命令行运行：

```powershell
python dnf/minimap_grid_preview.py
```

这个脚本不会调用 `Game.run()`，所以不会按键。

它只做：

- 找 DNF 窗口
- 截图
- 裁小地图
- 在右上角小地图位置画白色房间网格
- 显示放大的小地图调试窗口
- 显示 `P/Q/B/E/D`

## 6. 新增地图调试代码需要接入主程序吗

一般不需要。

推荐结构是：

```text
dnf/map_specs.py              # 地图配置，主程序和调试工具都会读
dnf/main.py                   # 真正刷图主程序
dnf/minimap_grid_preview.py   # 只看小地图准不准
dnf/minimap_nav.py            # 小地图识别、算方向、选门
```

也就是说：

- 新地图配置只放到 `map_specs.py`
- 主程序通过 `MiniMapNavigator` 自动读取
- 调试工具也通过 `MiniMapNavigator` 自动读取
- 不需要把调试脚本 import 到 `main.py`

如果把调试代码接进主程序，反而容易导致刷图时多开窗口、影响速度。

## 7. 调地图时看什么

运行 `dnf/minimap_grid_preview.py` 后，重点看这几个东西：

### 绿色框

绿色框是：

```python
crop_rect_800
```

它应该框住整个右上角小地图。

如果绿色框没盖住小地图，就调：

```python
crop_rect_800=(x1, y1, x2, y2)
```

### 白色网格

白色网格是：

```python
room_rect_800
```

它应该覆盖小地图里的房间点位区域。

如果白色网格偏左、偏右、偏上、偏下，就调：

```python
room_rect_800=(x1, y1, x2, y2)
```

### P / Q / B / E / D

这些是模板识别出来的图标：

- `P`：角色位置
- `Q`：问号房
- `B`：boss
- `E`：精英/特殊房
- `D`：下楼

如果 `P` 和 `Q` 没有压在真实图标上，方向就一定可能错。

比如真实问号在角色右边，但日志显示：

```text
current=(1, 0), query=(3, 0), direction=DOWN
```

说明程序认为问号在角色下面，这通常是：

- 白色网格区域不准
- 问号模板匹配错了
- 当前地图 rows/cols 不对

## 8. 新增地图完整流程

### 第一步：截图确认小地图位置

先用游戏当前分辨率截图，比如 800x600。

看右上角小地图大概范围。

### 第二步：新增 MapSpec

在 `dnf/map_specs.py` 里新增：

```python
"你的地图名": MapSpec(
    name="你的地图名",
    crop_rect_1067=(0, 0, 1, 1),
    crop_rect_800=(小地图左, 小地图上, 小地图右, 小地图下),
    minimap_width=小地图右 - 小地图左,
    minimap_height=小地图下 - 小地图上,
    rows=行数,
    cols=列数,
    room_grid=_all_walkable(行数, 列数),
    room_rect_800=(网格左, 网格上, 网格右, 网格下),
),
```

如果暂时不知道 1067 分辨率的小地图位置，可以先保留一个大概值，重点先调 `crop_rect_800` 和 `room_rect_800`。

### 第三步：用预览工具调

打开：

```text
dnf/minimap_grid_preview.py
```

把：

```python
MAP_NAME = "haibolun"
```

改成你的地图名。

运行后看绿色框和白色网格。

### 第四步：确认 P/Q 坐标

如果角色和问号在同一行左右相邻，日志应该类似：

```text
current=(3, 0)
query=(3, 1)
direction=RIGHT
```

如果显示：

```text
current=(1, 0)
query=(3, 0)
direction=DOWN
```

说明网格还是不准。

### 第五步：主程序运行

调准后运行主程序：

```powershell
$env:DNF_MAP_NAME="haibolun"
python -m dnf.main
```

如果还想开主程序里的小地图调试窗口：

```powershell
$env:DNF_DEBUG_MINIMAP="1"
$env:DNF_MAP_NAME="haibolun"
python -m dnf.main
```

## 9. 如果想让 auto 自动识别新地图

`DNF_MAP_NAME="auto"` 时，程序会在 `dnf/minimap_nav.py` 里对 `MAP_SPECS` 里的地图做评分。

新增到 `MAP_SPECS` 后，理论上 auto 会参与比较。

但实际建议：

```powershell
$env:DNF_MAP_NAME="haibolun"
```

先强制指定地图。

等这个地图调准后，再考虑用 `auto`。

因为 `auto` 依赖模板匹配分数，如果两个地图的小地图区域很像，可能会自动切错。

## 10. 最容易出错的地方

### 1. crop_rect_800 写成 None

如果你是 800x600 分辨率，不建议写：

```python
crop_rect_800=None
```

最好明确写出 800 分辨率的小地图裁剪区域。

### 2. room_rect_800 没写

如果不写，程序会把整个裁剪小地图当成房间网格。

这经常会把背景、边框也算进去，导致 `current/query` 行列错。

### 3. rows/cols 不对

比如实际是 4 行 5 列，但写成 5 行 9 列，就会直接算错方向。

### 4. P/Q 模板匹配错

如果白色网格没问题，但 `Q` 压不到真实问号上，就要调 `dnf/minimap_nav.py` 里的模板或阈值。

重点看：

```python
QUERY_TEMPLATE_FILES
MARKER_THRESHOLDS
```

## 11. 推荐调试顺序

推荐按这个顺序来，不要一开始就改方向算法：

1. 先让绿色框完整盖住小地图。
2. 再让白色网格盖住房间格子区域。
3. 再确认 `P` 压在角色蓝点上。
4. 再确认 `Q` 压在问号上。
5. 再看日志里的 `current/query` 行列。
6. 最后再看 `direction` 是否正确。

只要 `P/Q` 位置正确，`rows/cols` 正确，`room_rect_800` 正确，方向一般就会正确。
